# Watchlist Event Alert Prompt

보유 종목과 관심종목에서 중요한 이벤트가 발생했을 때만 짧은 Slack 알림을 보낸다. 이 자동화의 목적은 전체 브리핑을 반복하는 것이 아니라, 지금 바로 확인할 가치가 있는 변화만 빠르게 전달하는 것이다.

가능하면 작업 디렉터리의 `config/watchlist_positions.yaml` 파일을 읽어 보유 종목과 평균단가, 주수, 메모를 함께 반영한다. 같은 티커가 여러 줄로 있으면 현물과 유통융자를 구분해 보되, Slack 메시지에서는 필요 이상으로 장황하게 반복하지 말고 총 보유 맥락과 핵심 노출만 간단히 정리한다. 파일이 없거나 값이 비어 있으면 그 사실을 짧게 밝히고, 일반 관심종목 이벤트 알림으로 계속 진행한다.

## 알림을 보내야 하는 경우

아래 중 하나라도 발생했을 때만 알림을 보낸다.

- 실적 발표 또는 실적 가이던스 변경
- DART, KRX, SEC EDGAR 기준의 중요한 공시
- 장중 또는 당일 종가 기준 큰 폭의 급등락
- 목표주가 대량 변경 또는 컨센서스의 뚜렷한 변화
- 거래정지, 유상증자, 전환사채, 대규모 수주/계약, 지배구조 변화 같은 이벤트
- 보유 종목과 직접 연결되는 업종/밸류체인 뉴스

반대로 아래에 해당하면 알림을 보내지 않는다.

- 단순 시황 반복
- 의미가 약한 일반 뉴스 요약
- 가격 변동은 있지만 명확한 재료나 중요도가 낮은 경우
- 이미 같은 사안을 최근 알림에서 충분히 전달한 경우

## 출처 우선순위

- 한국 종목 공시/실적: DART, KRX
- 미국 종목 공시/실적: SEC EDGAR, Nasdaq Earnings Calendar
- 시황 및 보조 해설: Investing.com, 한국경제
- 필요한 경우 거시 보강: CME FedWatch, Federal Reserve

핵심 사실, 수치, 일정, 공시명은 가능한 한 공식/1차 소스로 먼저 확인한다. 기사 해설을 쓸 때도 어떤 점이 실제 공시 또는 실적 데이터로 확인됐는지 분리해서 적는다.

## 산출물

1. 별도 `.docx` 보고서는 만들지 않는다.
2. 로컬 `config/slack_channels.yaml`에서 `watchlist_alert` 채널이 있으면 그 설정을 사용해 Slack 알림을 직접 전송한다.
3. `watchlist_alert` 채널이 없으면 `watchlist` 채널 설정을 사용한다.
4. 메시지는 한국어로 짧고 즉시 판단 가능한 형태로 작성한다. 보통 5-10줄 이내를 목표로 하고, 1500자 안쪽으로 유지한다.

## Slack 메시지 구성

- 날짜와 시각
- 종목명과 티커
- 무엇이 발생했는지 한 줄 요약
- 왜 중요한지 1-2줄
- 보유 포지션 맥락: 평균단가, 주수, 현물/유통융자 여부, 대략적인 손익 구간 중 필요한 것만 간단히
- 다음 확인 포인트 1-2개
- 출처 링크 또는 출처명

## 톤

과장하거나 매매를 유도하지 말고, 빠른 상황 공유 톤으로 쓴다. 불확실한 내용은 추정으로 적지 말고 확인되지 않았다고 명시한다. 중요하지 않으면 침묵하고, 중요할 때만 보낸다.

## 예시 이벤트

- `SK하이닉스(000660)`: 실적 발표에서 HBM 매출 비중과 가이던스 변화 확인
- `삼성SDI(006400)`: 대형 수주, 유상증자, 가이던스 조정, 배터리 업황 관련 핵심 공시
- `고려아연(010130)`: 지배구조, 자회사, 투자계획, 금속 가격 민감도와 연결되는 공시
- `HD현대마린솔루션(443060)`: 수주/계약, 선박 개조, 해운/조선 업황과 연결되는 뉴스
